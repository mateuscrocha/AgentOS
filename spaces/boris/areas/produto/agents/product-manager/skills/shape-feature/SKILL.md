# Skill: shape-feature

## O que esta skill faz

Transforma uma necessidade de produto do Boris em definição funcional clara, com dor principal, usuário, fluxo, hipótese e critério de sucesso.

## Quando usar

- quando surgir uma ideia de feature, melhoria ou ajuste relevante de produto
- quando for preciso sair de pedido vago e chegar a uma definição de escopo utilizável

## Processo

1. Identificar a dor principal, o usuário e a decisão ou gargalo que a feature deve destravar.
2. Definir fluxo principal, comportamento esperado e limites do escopo.
3. Explicitar hipótese, trade-offs e critério de sucesso.
4. Entregar a definição funcional de forma prática.

## Inputs

- `$ARGUMENTS`: ideia de funcionalidade, dor percebida, usuário afetado, contexto operacional e qualquer restrição já conhecida

## Outputs

Definição de feature com:
- dor principal
- usuário
- problema a resolver
- fluxo principal
- hipótese
- critério de sucesso

## Regras

1. Se a dor principal não estiver clara, a feature ainda não está pronta para ser definida.
2. Escopo deve servir leitura e ação, não complexidade por si só.
3. Toda feature do Boris deve reforçar clareza operacional e confiança no uso.
