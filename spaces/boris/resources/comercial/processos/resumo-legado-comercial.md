# Resumo do Legado Comercial Boris

## O que veio de cada pasta

### `Boris - Enkrateia`

Contribui com a camada de metodo operacional:

- como organizar a base antiga
- como priorizar por lotes
- como definir prioridade P1, P2 e P3
- como conduzir rotina semanal de abordagem e follow-up
- como pensar uma base que faz agir

### `Boris - Leads`

Contribui com a camada de estrutura de dados:

- desenho de base mestra
- separacao entre dado bruto, limpo e consolidado
- campos minimos
- controle de origem
- modelo de status para operacao de leads

## Leitura recomendada

1. ler `boris-operacao-leads-enkrateia.md`
2. ler `boris-leads-operacao-legado.md`
3. revisar `boris-base-leads-template.csv`
4. consolidar o modelo final do pipeline do Boris

## Decisao sugerida

Usar os materiais assim:

- Enkrateia = metodo de organizacao e ataque da base
- Boris Leads = arquitetura da base e governanca de dados
- Area comercial atual = regra viva de pipeline, qualificacao e follow-up

## Pendencia principal

Definir um vocabulário final de status para evitar conflito entre:

- status do legado da Enkrateia
- status do desenho da base de leads
- estagios do `sales-operator`

## Recomendacao

O proximo passo de maior valor e criar um arquivo canonico de pipeline em `guidelines/` ou `processos/` unificando:

- status oficiais
- criterios de entrada e saida
- temperatura
- prioridade
- proximo passo esperado por estagio
