# Bóris Pose Variation Guide

Use this file to prevent Bóris from looking static or repeated across generations.

## Core Idea

Bóris must keep the same identity, silhouette family, face language, outline, and palette, but the pose should adapt to the scene instead of defaulting to the same raised-hand stance every time.

## Stable Identity

Keep stable:
- flame/drop silhouette family
- large black eyes with white highlight
- friendly simple smile language
- orange/yellow palette
- thick dark outline
- clean sides with no appendages

Allow variation in:
- arm position
- hand gesture
- torso angle
- head tilt
- leg stance
- weight distribution
- movement energy
- prop interaction
- expression intensity

## Approved Pose Families

- greeting: wave, soft wave, high wave, two-arm welcome
- presenting: one arm open, one finger pointing, both arms open, hand indicating a panel or object
- explaining: one hand up, one hand near chest, thoughtful gesture, subtle didactic pose
- working: typing on laptop, holding phone, checking screen, reading notebook, operating a tool
- carrying: holding mug, book, box, bag, gear, phone, magnifier, watering can
- walking: one leg forward, shopping stride, casual movement, forward lean
- celebrating: jump, raised fist, big open smile, energetic stance
- reacting: surprised, curious, thinking, investigative, attentive
- calm: seated, cross-legged, soft wave, relaxed neutral
- peeking: leaning from behind an edge, partial-body reveal

## Composition Rules

- Choose a pose that matches the story before adding decorative details.
- Avoid repeating the same hand-up front-facing stance unless the request is explicitly a neutral canonical pose.
- When generating a batch, vary the pose family across outputs.
- For thumbnails and campaign assets, prefer stronger silhouette readability and more energetic asymmetry.
- For transparent assets, prefer clean readable poses with one clear action.

## Props And Actions

Props are allowed when they support the use case and stay secondary to identity.

Good examples:
- mug
- book
- laptop
- phone
- box
- cloth
- watering can
- frying pan
- shopping bag
- magnifier
- gear

## Prompting Pattern

When you want variety, explicitly include a pose/action clause in the prompt:

- "use a new pose different from the default wave"
- "give Bóris a dynamic presenting pose"
- "show Bóris interacting naturally with the prop"
- "avoid the repeated front-facing raised-hand default pose"

## Avoid

- defaulting to the same exact waving pose for unrelated requests
- freezing both feet and torso in the same frontal neutral stance every time
- using props without meaningful hand interaction
- breaking silhouette readability with cluttered limbs
- changing the body design in order to create variation
